Exemple 1: `aria-label`

Aquest exemple mostra com utilitzar l'atribut aria-label per proporcionar una etiqueta personalitzada a un element.

<button class="btn" aria-label="Desa els canvis">💾</button>

Exemple 2: `aria-labelledby`

Aquest exemple mostra com utilitzar l'atribut aria-labelledby per associar un element amb una etiqueta externa.

Formulari de contacte

<h3 id="titol-formulari">Formulari de contacte</h3> <form aria-labelledby="titol-formulari"> <label for="nom">Nom:</label> <input type="text" id="nom" /> </form>

Exemple 3: `aria-describedby`

Aquest exemple mostra com utilitzar l'atribut aria-describedby per proporcionar una descripció addicional a un element.

Introdueix un correu electrònic vàlid.
<label for="email">Correu electrònic:</label> <input type="email" id="email" aria-describedby="ajuda-email" /> <div id="ajuda-email">Introdueix un correu electrònic vàlid.</div>

Exemple 4: `aria-hidden`

Aquest exemple mostra com utilitzar l'atribut aria-hidden per amagar un element a tots els productes de suport.

Valoració: 5 estrelles
<i class="fas fa-star" aria-hidden="true"></i> <span>Valoració: 5 estrelles</span>

Exemple 5: `aria-live`

Aquest exemple mostra com utilitzar l'atribut aria-live per anunciar canvis dinàmics en una regió de la pàgina.

<button class="btn" onclick="actualitzarMissatge()">Actualitza missatge</button> <div aria-live="polite" id="live-region" class="alert-box"></div> let comptador = 0; function actualitzarMissatge() { const liveRegion = document.getElementById('live-region'); comptador++; // Alterna el missatge afegint o traient un espai al final if (comptador % 2 === 1) { liveRegion.textContent = 'El contingut s\'ha actualitzat correctament!\u00A0'; } else { liveRegion.textContent = 'El contingut s\'ha actualitzat correctament!'; } liveRegion.hidden = false; }

Exemple 6: aria-required

aria-required="true" indica que un camp és obligatori, encara que no tingui l'atribut required HTML.

<input id="nom-req" aria-required="true" />

Exemple 7: aria-disabled

aria-disabled="true" indica que un element està desactivat per a tecnologia assistiva, encara que no tingui l'atribut disabled HTML. En aquest cas l'item no serà tabulable ja que deixa de ser interactuable.

Botó desactivat
<div class="btn" role="button" aria-disabled="true" tabindex="-1">Botó desactivat</div>

Exemple 8: aria-haspopup i role="menuitem" amb navegació per fletxes

aria-haspopup="true" indica que el botó obre un menú, diàleg o llista desplegable.
Si s'utilitza role="menuitem", cal permetre la navegació amb fletxes.

<button aria-haspopup="true" aria-controls="menu-exemple" aria-expanded="false">Mostra menú</button> <div id="menu-exemple" role="menu"> <button role="menuitem" tabindex="0">Opció 1</button> <button role="menuitem" tabindex="-1">Opció 2</button> <button role="menuitem" tabindex="-1">Opció 3</button> </div>

Exemple 9: aria-checked

aria-checked s'utilitza per indicar l'estat d'elements tipus checkbox, radio o botons toggle personalitzats.

<button role="checkbox" aria-checked="false" > <span class="checkbox-icon" aria-hidden="true"></span> <span>Desactivat</span> </button>
